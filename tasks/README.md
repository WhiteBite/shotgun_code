# Taskflow для Shotgun

Этот каталог описывает "конвейер задач" — как Shotgun эволюционно движется к целевой архитектуре.
Его читает Cursor по правилу .cursor/rules/70_taskflow.mdc (не трогай его, если уже есть).

Файлы:
- plan.yaml     — план задач (ид, имя, зависимости, бюджеты, путь к шагам)
- status.json   — текущее состояние задач (todo/in_progress/done/blocked)
- steps/*.yaml  — шаги/acceptance для задач (минимальные, без бюрократии)

Статусы:
- todo        — задача ещё не взята
- in_progress — выполняется
- done        — завершена
- blocked     — остановлена (нужна ручная помощь или внешние условия)
